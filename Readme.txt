Долгосрочное домашнее задание BY Vanya Ersh/Ilya Bool

Требования:
1.	Язык реализации модулей C#. Средство модульного тестирования NUnit. Интегрированная среда разработки Visual Studio 2013.
2.	Необходимо разработать трехуровневое приложение, включающее СУБД, сервер приложений, а также клиентское приложение.
	Клиенты посредством сервера приложений осуществляют передачу друг другу простых текстовых сообщений.
3.	В качестве СУБД рекомендуется использовать MySQL Community Server, а также библиотеку Connector/Net для доступа к нему из среды .NET.
	В качестве технологии доступа к базе данных рекомендуется использовать LINQ to Entities. Допускается развертывание СУБД на той же машине, что и сервер приложений. 
	К СУБД может осуществлять доступ только сервер приложений по инициативе клиента. Прямой доступ клиентских приложений к СУБД запрещен.
4.	Взаимодействие между клиентскими приложениями и сервером приложений должно осуществляться с использованием технологии Windows Communication Foundation (WCF) 
	через привязку на базе протокола TCP с использованием бинарной передачи данных. 
5.	Сервер приложений должен быть реализован в виде консольного приложения.
6.	Клиентские приложения должны иметь графический пользовательский интерфейс WinForms. 
7.	При запуске клиентского приложения пользователь вводит свое имя, который используется сервером для идентификации. 
	Имя высылается серверу приложений при каждом запросе. Проблемы безопасности, связанные с возможностью подделки клиентом своего имени, не рассматриваются.
8.	После подключения клиент выполняет запрос к серверу для получения сообщений, которые были отправлены ему другими клиентами в тот момент, 
	когда он не был подключен к серверу приложений. Сервер приложений выполняет выборку запрошенных сообщений из СУБД, отсылает их клиенту и после успешной отправки удаляет их из СУБД. 
9.	Клиент запрашивает у сервера приложений список всех клиентов и их текущий статус (находятся они в сети или нет). 
	Сервер приложений должен хранить в оперативной памяти список всех клиентов, которые выполняли к нему подключения. 
	Сервер приложений отвечает за определение факта отключения клиента.
10.	Клиент может отослать сообщение другому клиенту через сервер приложений, выполнив соответствующий запрос и указав идентификатор клиента,
	 которому необходимо отослать сообщение. Если адресат подключен к серверу, то ему немедленно высылается сообщение. 
	Если адресат в текущий момент не подключен к сети, то в СУБД производится сохранение сообщения для его последующей отправки клиента при его последующем подключении.
11.	Графический интерфейс клиентского приложения должен обеспечивать возможность просмотра списка клиентов, отображения их статуса, 
	редактирования и отправки сообщения выбранному абоненту, а так же получение сообщений.
12.	В системе должны быть выделены модули, относящиеся к сетевому взаимодействию, взаимодействию с СУБД и графическому интерфейсу пользователя. Модули должны быть оформлены в виде сборок.
